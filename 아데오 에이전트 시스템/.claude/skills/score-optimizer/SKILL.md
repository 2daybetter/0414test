# score-optimizer

## 개요

rfp-context의 평가항목 배점 테이블을 분석하여 아데오의 강점이 극대화되는 섹션별 강조 전략과 차별화 포인트를 도출한다.  
출력인 evaluation-strategy는 proposal-writer의 핵심 입력으로, 제안서 챕터 순서·분량·핵심 메시지를 결정한다.

**출력 형식 원칙**:
- **evaluation-strategy**: Google Drive MCP(`mcp__claude_ai_Google_Drive__create_file`)로 업로드
- 로컬 `.md` 저장 금지

---

## 실행 절차

### Step 1: 입력 확인

다음 항목이 모두 있는지 확인한다. 없으면 중단 후 누락 항목 명시.

| 필수 입력 | 확인 방법 |
|---------|---------|
| rfp-context Drive URL | `.status` 파일의 `outputs.rfp-context` |
| 프로젝트명 | `.status` 파일의 `lead` |

`.status` 파일 경로: Google Drive 아데오 프로젝트/{프로젝트명}/.status/.status (루트 폴더 ID: 1XHWdKpQmsoyiScj-NicRrHuYzDZsyBDM)

### Step 2: rfp-context 읽기

`mcp__claude_ai_Google_Drive__read_file_content`로 rfp-context를 읽고 다음 섹션을 추출한다:

1. **평가항목 배점 테이블** — 항목명, 배점, 세부 평가 기준 전체
2. **수행 요구사항** — 필수 기술 스택, 인력 요건, 일정 기준
3. **고객사 정보** — 사업 배경, 현황, 기대 효과

### Step 3: 배점 파싱 및 검증

추출한 배점 테이블을 파싱한다.

**검증 기준**:
- 배점 합계가 80점인지 확인 (기술능력 평가 기준)
- 항목이 최소 4개 이상인지 확인

**검증 결과 분기**:

| 결과 | 처리 |
|------|------|
| 배점 합계 80점, 항목 4개 이상 | Step 4로 진행 |
| 배점 테이블 미추출 | rfp-context 재파싱 1회 (Step 2 재실행) |
| 재파싱 후에도 없음 | 에스컬레이션 대기: "RFP 내 평가 배점 섹션 확인 요청" 출력 |
| 배점 합계 80점 미충족 | 추출된 항목만으로 진행, 미충족 항목 "RFP 미기재" 표기 |

### Step 4: 강조도 매핑

각 평가항목에 강조도(상/중/하)를 아래 기준으로 배정한다.

**강조도 기준**:

| 강조도 | 배점 비율 | 처리 방식 |
|-------|---------|---------|
| 상 | 배점 상위 30% (15점 이상) | 전용 챕터 독립 분리, 최다 지면 할당 |
| 중 | 배점 중위 50% (8~14점) | 주요 섹션에 상세 기술 |
| 하 | 배점 하위 20% (7점 이하) | 종합 섹션에 간략 언급 |

**더케이예다함 사업 기준 예시** (rfp-context 확인 후 실제값으로 대체):

| 평가항목 | 배점 | 강조도 |
|---------|------|-------|
| UI/UX 혁신성 | 15점 | 상 |
| 기술 아키텍처 | 15점 | 상 |
| 유사 실적 | 12점 | 상 |
| PM/일정 관리 | 10점 | 중 |
| 접근성·보안 | 10점 | 중 |
| 모바일 앱 | 8점 | 중 |
| 데이터 마이그레이션 | 5점 | 하 |
| 유지보수 | 5점 | 하 |

### Step 5: 차별화 포인트 도출

각 강조도(상) 항목에 대해 아데오의 차별화 포인트를 최소 3개씩 도출한다.

**도출 기준**:
1. 배점 세부 평가 기준 문장을 역분석하여 채점자 관점의 기대값 추출
2. 아데오의 기술 스택·수행 실적과 연결되는 강점 매핑
3. 경쟁사 대비 차별화가 가능한 포인트 우선 선정

**차별화 포인트가 0개인 경우**: 재실행 1회 → 여전히 0개이면 이사님 판단 요청.

### Step 6: evaluation-strategy 문서 작성

다음 구조로 evaluation-strategy 문서를 작성한다.

```markdown
# Evaluation Strategy — {프로젝트명}

## 평가항목 배점 요약

| 항목 | 배점 | 강조도 |
|------|------|-------|
| {항목명} | {점수}점 | 상/중/하 |
...
| 합계 | 80점 | — |

## 강조도(상) 항목 — 전략

### {항목명} ({점수}점)
- **세부 평가 기준**: {rfp-context 원문}
- **차별화 포인트**:
  1. {포인트 1}
  2. {포인트 2}
  3. {포인트 3}
- **제안서 챕터 배치**: {권장 챕터 위치}
- **핵심 메시지**: {한 문장 요약}

...

## 강조도(중) 항목 — 전략

### {항목명} ({점수}점)
- **세부 평가 기준**: {rfp-context 원문}
- **핵심 어필 포인트**: {주요 강조 내용}
- **제안서 섹션 배치**: {권장 섹션}

...

## 강조도(하) 항목 — 간략 처리

| 항목 | 언급 방식 | 배치 위치 |
|------|---------|---------|
| {항목명} | {간략 언급 방식} | 종합 섹션 |
...

## 제안서 챕터 순서 권고

1. {강조도(상) 항목 기반 챕터}
2. ...

## 채점자 포인트 체크리스트

- [ ] 강조도(상) 항목별 전용 챕터 존재
- [ ] 차별화 포인트 각 항목 3개 이상 명시
- [ ] 배점 비중 높은 항목이 앞 챕터에 배치
```

### Step 7: Google Drive 업로드

생성한 evaluation-strategy 문서를 Google Drive MCP로 업로드한다.

**실행 순서**:
1. `mcp__claude_ai_Google_Drive__create_file`로 `evaluation-strategy_{프로젝트명}_{YYYYMMDD}.md` 업로드
2. 반환된 Drive URL을 `.status` 파일의 `outputs.evaluation-strategy`에 기록

### Step 8: 검증

**성공 기준**:
- 배점 합계 80점 검증 완료 (또는 미충족 항목 "RFP 미기재" 표기)
- 강조도(상) 항목 각각 차별화 포인트 3개 이상
- 제안서 챕터 순서 권고 포함
- `.status` 파일의 `outputs.evaluation-strategy`에 Drive URL 기록 완료

| Drive 업로드 실패 | 재시도 1회 → 실패 시 에스컬레이션 |
