# wireframe-spec

## 개요

IA 설계서 기반으로 아데오 표준 화면설계서(DE-03)를 생성한다.  
반드시 `/templates/wireframe-template.md` 를 참조하여 작성한다.  
IA의 **Page 타입 화면 전체**를 커버해야 하며, 단 하나의 화면도 누락 불가.

---

## 실행 절차

### Step 1: 입력 확인

| 필수 입력 | 확인 방법 |
|---------|---------|
| IA 설계서 경로 | `/output/구축사/{프로젝트명}/웹기획팀/ia-{프로젝트명}.md` 존재 여부 |
| 화면설계서 템플릿 | `/templates/wireframe-template.md` 존재 여부 |
| 고객사 브랜드 가이드 | 있으면 참조, 없으면 기본값 사용 |

### Step 2: Page 타입 화면 목록 추출

IA 설계서에서 `Type = Page` 인 화면을 모두 추출한다.  
추출된 화면 수가 화면설계서 커버 기준이 된다.

### Step 3: 화면별 설계서 작성

추출된 각 Page 화면에 대해 다음 4개 블록을 순서대로 작성한다.

#### 3-1. 화면 헤더

```
| 항목 | 내용 |
|------|------|
| 화면ID | {FO_NNN / BO_NNN} |
| 화면명 | {화면 이름} |
| URL 경로 | {URL} |
| 버전 | v1.0 |
| 연관 화면ID | {팝업·레이어 등 관련 화면} |
| 작성일 | {YYYY-MM-DD} |
```

#### 3-2. ASCII 레이아웃

텍스트 기반 화면 구성 표현. 실제 디자인이 아닌 구성 요소 배치를 나타낸다.

```
+--[GNB: 로고 | 메뉴1 | 메뉴2 | 로그인]--+
|                                        |
|  [히어로 배너]                          |
|  제목 텍스트 / CTA 버튼                  |
|                                        |
|  [섹션1: 카드 리스트]                    |
|  [카드] [카드] [카드]                   |
|                                        |
+--[Footer: 주소 | 연락처 | SNS]----------+
```

상세 규칙은 `references/wireframe-rules.md` 참조.

#### 3-3. 기능 설명 테이블

```
| 번호 | 구성요소 | 유형 | 기능 설명 | 비고 |
|------|---------|------|---------|------|
| 1 | GNB | 내비게이션 | 메뉴 클릭 시 해당 페이지 이동 | 고정(sticky) |
| 2 | 히어로 배너 | 이미지+텍스트 | 자동 슬라이드 (3초 간격) | BO 관리 |
```

#### 3-4. 화면 조건

```
| 조건 구분 | 내용 |
|---------|------|
| 로그인 전 | {비로그인 시 표시/숨김 항목} |
| 로그인 후 | {로그인 후 변경 사항} |
| PC (1280px+) | {PC 전용 레이아웃 특이사항} |
| Mobile (375px~) | {모바일 레이아웃 차이} |
| 권한별 분기 | {관리자/일반 사용자 등 권한 차이, 해당 시} |
```

### Step 4: 공통 컴포넌트 정의

화면설계서 맨 앞에 다음 공통 컴포넌트를 정의한다:

| 컴포넌트 | 내용 |
|---------|------|
| GNB (Global Navigation Bar) | 전체 페이지 공통 상단 내비게이션 구조 |
| Footer | 전체 페이지 공통 하단 구조 |
| 공통 모달/팝업 | 알림, 확인, 오류 등 공통 팝업 |
| 로딩 상태 | 데이터 로딩 중 UI |
| 빈 상태 | 데이터 없을 때 UI |

### Step 5: 자기 검증

모든 화면 작성 후 다음을 자체 확인한다:

- [ ] IA의 모든 Page 타입 화면이 커버되었는가
- [ ] 각 화면에 헤더·ASCII 레이아웃·기능 설명 테이블·화면 조건 4항목이 모두 있는가
- [ ] 공통 컴포넌트 섹션이 있는가

검증 실패 항목이 있으면 해당 화면만 재작성 후 재확인.

### Step 6: 출력 및 저장

- **출력 경로**: `/output/구축사/{프로젝트명}/웹기획팀/화면설계서-{프로젝트명}.md`
- **성공 기준**: IA 전체 Page 화면 커버 + 각 화면 4항목 완비 + 공통 컴포넌트 섹션 존재

## 실패 처리

| 실패 유형 | 처리 |
|---------|------|
| IA 설계서 없음 | 실패 리포트 생성 후 중단 |
| 화면 누락 발견 (자기 검증) | 해당 화면 즉시 보완 후 재검증 |
| 자기 검증 2회 실패 | 미완성 화면 목록 명시 후 에스컬레이션 |
