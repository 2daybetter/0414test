# 화면설계서 작성 규칙

## ASCII 레이아웃 작성 규칙

### 레이아웃 표현 기호

| 기호 | 의미 |
|------|------|
| `+---+` | 영역 경계 (상단/하단) |
| `\|` | 영역 경계 (좌/우) |
| `[컴포넌트명]` | UI 컴포넌트 |
| `{내용 설명}` | 동적 데이터 영역 |
| `...` | 반복 요소 |
| `(버튼명)` | 클릭 가능한 버튼 |

### 레이아웃 세부 표현 예시

**리스트 페이지**:
```
+--[GNB]------------------------------------------+
|                                                  |
|  [페이지 타이틀]  [필터: 전체▼] [정렬: 최신순▼]    |
|                                                  |
|  [카드] [카드] [카드]                             |
|  [카드] [카드] [카드]                             |
|                                                  |
|  [페이지네이션: < 1 2 3 ... >]                    |
|                                                  |
+--[Footer]---------------------------------------+
```

**폼 페이지**:
```
+--[GNB]------------------------------------------+
|                                                  |
|  [브레드크럼: 홈 > 서비스 > 신청]                  |
|                                                  |
|  [섹션 타이틀]                                    |
|  이름 [____________________]                     |
|  이메일 [____________________]                   |
|  내용 [____________________]                     |
|       [____________________]                     |
|                                                  |
|  (취소)  (제출)                                   |
|                                                  |
+--[Footer]---------------------------------------+
```

**상세 페이지**:
```
+--[GNB]------------------------------------------+
|                                                  |
|  [브레드크럼]                                     |
|                                                  |
|  [이미지 영역 / 갤러리]                            |
|                                                  |
|  [타이틀]                                        |
|  [메타정보: 날짜 | 카테고리 | 작성자]               |
|                                                  |
|  [본문 콘텐츠]                                    |
|                                                  |
|  [관련 콘텐츠 목록]                               |
|                                                  |
+--[Footer]---------------------------------------+
```

## 기능 설명 테이블 작성 기준

### 구성요소 유형 분류

| 유형 | 예시 |
|------|------|
| 내비게이션 | GNB, 탭, 브레드크럼, 페이지네이션 |
| 이미지+텍스트 | 배너, 히어로, 카드 |
| 폼 요소 | 입력 필드, 드롭다운, 체크박스, 라디오 |
| 버튼 | 제출, 취소, 더보기, 공유 |
| 리스트 | 게시판, 상품목록, 카드리스트 |
| 팝업/모달 | 알림 팝업, 레이어 팝업 |
| 미디어 | 이미지, 동영상, 슬라이더 |
| 정보 표시 | 테이블, 차트, 태그, 뱃지 |

### 기능 설명 작성 기준

- **명확한 동작 동사 사용**: "클릭 시 ~", "입력 시 ~", "스크롤 시 ~"
- **연결 화면 명시**: "클릭 시 {화면ID} 이동" 형식
- **유효성 검증**: 폼 요소는 유효성 규칙 명시 (예: "필수입력, 이메일 형식 검증")
- **BO 연동 명시**: BO에서 관리되는 콘텐츠는 "BO {화면ID}에서 관리" 기재

## 화면 조건 작성 기준

### 로그인 분기

| 케이스 | 작성 내용 |
|--------|---------|
| 로그인 전 | 숨겨지거나 비활성화되는 기능 명시 (예: "북마크 버튼 비활성 — 클릭 시 로그인 유도 팝업") |
| 로그인 후 | 추가로 표시되는 기능 명시 (예: "마이페이지 링크 활성화") |

### 반응형 분기

| 브레이크포인트 | 기준 | 주요 변경 사항 |
|-------------|------|-------------|
| PC | 1280px 이상 | 기본 레이아웃 |
| Tablet | 768px ~ 1279px | 2컬럼 → 1컬럼 전환 등 |
| Mobile | 375px ~ 767px | GNB → 햄버거 메뉴, 카드 세로 배치 등 |

### 권한 분기

관리자·일반사용자·비회원 등 권한에 따라 표시 여부가 다른 경우만 명시.

## 공통 컴포넌트 정의 기준

최소한 다음 5가지를 정의한다:

1. **GNB** — 로고, 메뉴 목록, 로그인/로그아웃 버튼, 모바일 햄버거 메뉴
2. **Footer** — 회사명, 주소, 연락처, 저작권, SNS 링크, 패밀리사이트
3. **공통 알림 모달** — 성공/오류/확인 3종
4. **공통 로딩 스피너** — 데이터 로딩 중 표시
5. **공통 빈 상태** — 데이터 없을 때 안내 문구 + 일러스트
