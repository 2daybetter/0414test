## 업무이력DB 구조

### Google Sheet 탭 구성

| 탭명 | 대상 파트 |
|------|---------|
| 비서실 | 비서실 직접 업무 |
| 제안파트 | 영업팀·전략팀·정책자금팀 |
| 구축파트 | PM·웹기획팀·디자인팀·개발팀 |

| 연구소 | 기술 리서치·R&D |

### 각 탭 컬럼 정의

| 컬럼 | 타입 | 설명 |
|------|------|------|
| 번호 | 숫자 | 자동 순번 |
| 파트 | 텍스트 | 비서실/제안파트/구축파트/운영파트/연구소 |
| 업무명 | 텍스트 | 업무 제목 (간결하게) |
| 담당자 | 텍스트 | 담당 팀원 이름 |
| 시작일 | 날짜 (YYYY-MM-DD) | 업무 시작일 |
| 완료예정일 | 날짜 (YYYY-MM-DD) | 완료 목표일 |
| 진행률 | 숫자 (0~100) | % 단위 |
| 상태 | 드롭다운 | 예정/진행중/완료/보류/취소 |
| 주요내용 | 텍스트 | 업무 내용 요약 |
| 커뮤니케이션이력 | 텍스트 | "YYYY-MM-DD 내용" 형식, 줄바꿈 구분 |
| 비고 | 텍스트 | 참고 사항 |

## 주간보고 탭 명명 규칙

```
{YYYY-MM-DD} 주간
예: 2026-04-20 주간
```

- 항상 해당 주 **월요일** 날짜 기준
- 동일 날짜 탭 존재 시: `2026-04-20 주간(수정)` 추가

---

## 업무 분류 기준

### 금주 완료 업무

- `상태 = "완료"` AND `완료예정일 ≥ 전주월요일`
- 또는 커뮤니케이션이력에 금주 날짜 완료 기록 포함

### 금주 진행 업무 (중요도 정렬)

1. `상태 = "진행중"` 전체
2. `진행률` 기준 내림차순 → 완료 임박 순서로 상단 노출
3. 최대 10건 표시 (초과 시 요약)

### 차주 예정 업무

- `상태 = "예정"` AND `시작일 ≤ 차주금요일`
- 또는 `상태 = "진행중"` AND `완료예정일 ∈ 차주`

### 지연/이슈 기준

| 조건 | 처리 |
|------|------|
| `완료예정일 < 오늘` AND `상태 ≠ "완료"` | 지연 표시 ⚠️ |
| `진행률 = 0` AND `시작일 < 오늘 -3일` | 착수 지연 표시 |
| `상태 = "보류"` | 보류 사유 표시 |

## 커뮤니케이션 이력 파싱 규칙

DB의 `커뮤니케이션이력` 컬럼 파싱:

```
형식: YYYY-MM-DD [유형] 내용
유형: 회의/통화/메일/메신저/기타
예: 2026-04-18 [회의] 킥오프 미팅 진행 — 주요 결정사항 정리
```

주간보고에는 **금주(월~일) 발생** 이력만 포함.  
이력이 없는 파트는 "이번 주 커뮤니케이션 없음" 으로 표기.

---

## 보고서 서술 원칙

1. 수치만 존재하는 경우 추정 표현("예상됩니다") 사용 금지
2. 지연 항목은 별도 섹션으로 반드시 분리
3. 0건 항목도 생략하지 않고 "없음" 표기
4. 커뮤니케이션 이력은 요약 1줄로 압축
