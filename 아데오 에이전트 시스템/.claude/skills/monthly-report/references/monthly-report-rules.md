# 월간보고 작성 규칙

## 집계 기간 정의

| 항목 | 기준 |
|------|------|
| 보고 대상 월 | 실행일 기준 직전 달 |
| 집계 시작 | 직전 달 1일 00:00 |
| 집계 종료 | 직전 달 말일 23:59 |
| 비교 기간 | 전전 달 (전월 대비 증감) |

---

## 업무이력DB 필터 기준

### 완료 업무

```
상태 = "완료"
AND (완료예정일 ∈ 집계월 OR 커뮤니케이션이력에 집계월 완료 기록)
```

### 진행 중 업무

```
상태 = "진행중"
AND 시작일 ≤ 집계월 말일
```

### 신규 착수

```
시작일 ∈ 집계월
```

### 지연 항목

```
완료예정일 < 집계월 말일
AND 상태 ≠ "완료"
AND 상태 ≠ "취소"
AND 상태 ≠ "보류"
```

---

## 탭 명명 규칙

```
{YYYY-MM} 월간
예: 2026-04 월간
```

- 직전 달 기준 (실행일 2026-05-01 → "2026-04 월간")
- 동일 탭 존재 시: `2026-04 월간(수정)` 추가

## KPI 정의 및 목표값

| KPI | 계산 | 목표 | 경고 |
|-----|------|------|------|
| 업무 완료율 | 완료/(완료+진행+보류) × 100 | 70% 이상 | 50% 미만 |
| 지연 발생률 | 지연건수/전체 × 100 | 10% 이하 | 20% 초과 |
| 파트별 착수율 | 신규착수/계획예정 × 100 | 80% 이상 | 60% 미만 |
| 커뮤니케이션 건수 | 이력 항목 수 합계 | — | 0건 시 알림 |

달성 기준: ✅ 목표 달성 / ⚠️ 경고 미달 / ❌ 심각 미달

## 파트별 집계 항목

### 구축파트 프로젝트 현황

업무명이 프로젝트 코드(`SKB-YYYY-NNN`) 형식인 항목을 프로젝트로 처리:

| 컬럼 | 집계 방법 |
|------|---------|
| 단계 | 진행률 구간으로 추정 (0~10%: PM, 11~30%: AY, 31~60%: DE, 61~90%: IM, 91~99%: TE, 100%: OP) |
| 진행률 | DB `진행률` 컬럼 직접 사용 |
| 납기 | DB `완료예정일` 사용 |
| 상태 | 정상(납기 여유 >14일)/지연위험(7~14일)/지연(7일 미만) |

### 제안파트 활동 현황

제안파트 탭에서:
- `업무명` 에 "제안" 포함 → 제안 건수
- `상태 = "완료"` AND `비고` 에 "수주" 포함 → 수주 건수
- `비고` 에 "수주금액: N원" 형식 → 수주 금액 집계

## 커뮤니케이션 이력 집계

DB `커뮤니케이션이력` 컬럼 형식:
```
YYYY-MM-DD [유형] 내용
```

월간보고에는 **집계월 발생 전체** 이력을 파트별로 그룹화:
- 동일 날짜·유형 이력은 1줄로 요약
- 10건 초과 시 "주요 N건" + 상세 별첨 표시

---

## 보고서 서술 원칙

1. **Executive Summary (핵심 요약)**: 전 파트 통합 3~5줄, 가장 중요한 성과·이슈 중심
2. 확인되지 않은 수치 추정 금지 ("예상됩니다" 표현 사용 시 반드시 "추정" 명시)
3. 0건 항목 생략 금지 ("활동 없음" 명시)
4. 지연 항목은 원인과 조치 방향을 함께 기술
5. 다음 달 계획은 DB `상태=예정` 항목 기반으로 작성
