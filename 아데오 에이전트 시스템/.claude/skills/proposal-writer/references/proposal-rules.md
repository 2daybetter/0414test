# 제안서 작성 규칙

## 섹션별 작성 기준

### 경영진 요약 (Executive Summary)

- **분량**: 전체 제안서의 10% 이하, 최대 2페이지 (마크다운 기준 최대 50줄)
- **구조**: 문제 인식 → 솔루션 → 기대 효과 순
- **문체**: 발주처 의사결정자(비전문가) 관점에서 쉽게 작성
- **필수 포함**: 핵심 메시지 1문장, 기대 효과 3가지, 수행 기간

### 사업 이해

- RFP에서 추출한 발주처 핵심 니즈를 그대로 인용 후 해석
- "발주처가 원하는 것" → "아데오가 제공할 것" 구조로 작성
- 잘못된 이해 방지: RFP에 없는 내용 추가 금지

### 수행 방법론

| 단계 | 표기 방식 |
|------|---------|
| 착수 | PM (Kick-off) |
| 분석 | AY (Analysis) |
| 설계 | DE (Design) |
| 구현 | IM (Implementation) |
| 테스트 | TE (Test) |
| 오픈 | OP (Open) |

각 단계별 주요 활동 및 산출물을 1~2문장으로 기재.

### 팀 구성 작성 기준

| 역할 | 포함 내용 |
|------|---------|
| PM | 이름(이니셜), 경력 연수, 유사 프로젝트 수행 실적 |
| 웹기획 | 이름(이니셜), 경력 연수 |
| 디자인 | 이름(이니셜), 경력 연수 |
| 개발 | 이름(이니셜), 역할(FE/BE), 경력 연수 |

개인 정보 보호를 위해 성명 전체 대신 이니셜 또는 직급만 사용.

### 차별화 포인트 작성 기준

- **3가지 이상** 반드시 작성
- **형식**: "경쟁사 대비 [우리의 강점]이 [구체적 수치 또는 방법]으로 [기대 효과]"
- 추상적 표현 금지 ("최고의 품질" → "동종 업계 평균 대비 결함률 30% 낮음" 등)

### 레퍼런스 (수행 실적) 작성 기준

| 항목 | 내용 |
|------|------|
| 고객사명 | 실명 또는 "A사(동종 업계)" |
| 프로젝트 유형 | 홈페이지 구축, 플랫폼 개발 등 |
| 수행 기간 | YYYY-MM ~ YYYY-MM |
| 주요 특징 | 이번 RFP와 유사한 요소 1~2개 |

## Figma 슬라이드 디자인 시스템

> 레퍼런스 Figma: https://www.figma.com/design/ySSfVAluSdMIfbn7vpvZbz/제안서_샘플?node-id=1-1556

### 슬라이드 기본 규격

| 항목 | 값 |
|------|---|
| 캔버스 크기 | W=1920px, H=1080px (16:9 프레젠테이션) |
| 폰트 패밀리 | Pretendard Variable (Regular / Medium / Bold) |
| 배경 | #FFFFFF (흰 배경) |

### 헤더

| 영역 | 내용 | 스타일 |
|------|------|------|
| 좌측 | 2depth 제목만 (프로젝트명 생략) | Bold-40px, 텍스트 컬러 |
| 우측 | 1depth 섹션명 | Medium-24px, 그레이 (#6B7580), 우측 정렬 |
| 구분선 | Y=74 | 1px, 연한 그레이 (#E0E2EA) |

### 캐치프레이즈

| 항목 | 값 |
|------|---|
| 위치 | 상단 중앙, 1행 Y=220 / 2행 Y=300 |
| 1행 크기 | Bold-68px |
| 2행 크기 | Bold-68px |
| 정렬 | 가운데 정렬 |
| 1행 색상 | 텍스트 기본색 (#1A1E2B) |
| 2행 색상 | 포인트 블루 (#256EF4) |

### 그레이 소제목 (선택)

- 필요 시에만 출력 — **Medium-40px**, 그레이, 가운데 정렬, Y=428
- 소제목 있을 때 콘텐츠 영역 시작: Y=502 / 없을 때: Y=430

### 콘텐츠 영역

| 요소 | 스타일 |
|------|------|
| 카드 제목 | Bold-32px, 흰색 |
| 카드 설명 | Medium-16px, 연한 그레이 (#8F99A8) |
| 아이콘/레이블 | Bold-22px, 연한 블루 (#99C2FF) |
| 다크 카드 배경 | #2A2B30, 모서리 radius=16px |
| 3열 카드 폭 | ~334px |
| 4열 카드 폭 | ~245px |

### 풋터

| 항목 | 값 |
|------|---|
| 구분선 | Y=H-38, 1px 연한 그레이 |
| 텍스트 | "블라인드 평가용", 12px, 그레이, 우측 정렬 |
| 회사명/로고 | **표시 금지** (블라인드 평가) |

## 금지 사항

- 발주처 명칭 오기 (RFP에 표기된 공식 명칭 그대로 사용)
- 확인되지 않은 수치 또는 실적 기재
- 경쟁사 직접 비방 표현
- 계약 금액 또는 원가 구조 노출
