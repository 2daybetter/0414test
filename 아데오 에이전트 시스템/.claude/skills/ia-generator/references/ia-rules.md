# IA 설계 규칙

## IA_ID 체계

| 구분 | 형식 | 예시 | 설명 |
|------|------|------|------|
| FO | FO-N-NN | FO-1-01 | N = 1depth 번호, NN = 화면 순번 (01~99) |
| BO | BO-N-NN | BO-1-01 | 동일 규칙 적용 |

## 화면 타입 정의

| Type | 정의 | URL 필요 | SEO 필요 |
|------|------|---------|---------|
| Page | 독립 URL을 가진 전체 화면 | ✅ | ✅ (FO만) |
| Layer Popup | 현재 화면 위에 레이어로 표시 | ❌ | ❌ |
| Popup | 별도 팝업 창으로 표시 | ❌ | ❌ |

## URL 설계 규칙

- 소문자 + 하이픈(-) 또는 슬래시(/) 사용
- 언더스코어(_) 금지 (SEO 불리)
- FO: `/경로/하위경로` (예: `/products/detail`)
- BO: `/admin/경로` (예: `/admin/products/list`)
- 동적 파라미터: `/경로/{id}` 형식

## SEO 메타 기준

| 항목 | 작성 기준 | 길이 제한 |
|------|---------|---------|
| Title | 페이지 핵심 키워드 + 브랜드명 | 30자 이내 |
| Description | 페이지 내용 요약 + CTA | 80자 이내 |
| Keyword | 쉼표 구분, 주요 검색어 | 5개 이내 |

## 1depth 메뉴 가이드라인

### FO 1depth 공통 구성 예시

| 업종 | 1depth 예시 |
|------|-----------|
| 기업 홈페이지 | 회사소개 / 사업영역 / 솔루션 / 뉴스&미디어 / 채용 |
| 쇼핑몰 | 홈 / 카테고리 / 이벤트 / 마이페이지 / 고객센터 |
| 서비스 플랫폼 | 서비스소개 / 이용방법 / 요금제 / 고객지원 |

### BO 1depth 공통 구성 예시

| 기능 | 설명 |
|------|------|
| 콘텐츠관리 | 게시판, 팝업, 배너 등 FO 콘텐츠 관리 |
| 회원관리 | 회원 정보, 권한, 탈퇴 처리 |
| 통계/분석 | 방문자, 전환율, 콘텐츠 조회수 |
| 시스템관리 | 메뉴 구성, 코드 관리, 접근 권한 설정 |
