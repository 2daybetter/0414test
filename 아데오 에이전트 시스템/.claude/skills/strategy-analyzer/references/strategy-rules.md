# 요구사항 정의 작성 규칙

## 요구사항 코드 체계

| 구분 | 형식 | 예시 | 설명 |
|------|------|------|------|
| 기능 요구사항 | REQ-F-NNN | REQ-F-001 | 사용자가 직접 사용하는 기능 |
| 비기능 요구사항 | REQ-NF-NNN | REQ-NF-001 | 성능·보안·가용성 등 품질 속성 |
| 제약사항 | REQ-C-NNN | REQ-C-001 | 기술·일정·법적 제약 |

순번은 001부터 시작하며 FO/BO 구분 없이 연번으로 부여한다.

## 우선순위 정의

| 등급 | 기호 | 기준 |
|------|------|------|
| 필수 | Must | 없으면 서비스 불가 — 반드시 구현 |
| 권장 | Should | 없어도 운영 가능하나 품질에 영향 |
| 선택 | Nice | 있으면 좋으나 예산/일정에 따라 제외 가능 |

## 비기능 요구사항 작성 기준

### 성능 기준 (기본값)

| 항목 | 기준 | 비고 |
|------|------|------|
| 페이지 응답시간 | 3초 이내 (일반 페이지) | 이미지 많은 페이지 5초 이내 |
| API 응답시간 | 1초 이내 | 검색 2초 이내 허용 |
| 동시접속자 | 최소 100명 기준 | 고객사 요구에 따라 조정 |
| 가용성 | 99.5% 이상 | 연간 다운타임 44시간 이내 |

### 보안 기준 (기본값)

| 항목 | 기준 |
|------|------|
| 인증 | 세션 기반 또는 JWT |
| 비밀번호 | 8자 이상, 영문+숫자+특수문자 조합 |
| HTTPS | 전체 페이지 SSL/TLS 적용 |
| 개인정보 | 암호화 저장 (AES-256 이상) |
| XSS/SQL Injection | 입력값 검증 및 이스케이프 처리 |

### 호환성 기준 (기본값)

| 구분 | 지원 범위 |
|------|---------|
| 브라우저 | Chrome 최신 2버전, Edge 최신 2버전, Safari 최신 2버전 |
| 모바일 OS | iOS 15+, Android 10+ |
| 반응형 기준 | PC 1280px+, Tablet 768px~1279px, Mobile 375px~767px |

## 기능 요구사항 테이블 필수 컬럼

| REQ_ID | 구분 | 기능명 | 기능 설명 | 우선순위 | 비고 |
|--------|------|-------|---------|---------|------|

- **REQ_ID**: REQ-F-NNN 형식
- **구분**: FO / BO
- **기능명**: 간결한 기능 이름 (예: 회원 로그인, 게시판 목록)
- **기능 설명**: 사용자 관점의 기능 설명 (1~2문장)
- **우선순위**: Must / Should / Nice
- **비고**: 관련 시스템 연동, 법적 요건 등 특이사항

## FO 공통 기능 체크리스트

기업 홈페이지 프로젝트 시 아래 항목이 요구사항에 포함되었는지 확인:

- [ ] 메인 페이지 (인트로, 주요 메뉴 CTA)
- [ ] 회사 소개 (비전, 연혁, 조직도)
- [ ] 서비스/사업 소개
- [ ] 뉴스·공지사항 게시판
- [ ] 채용 공고 (있는 경우)
- [ ] 고객 문의 폼
- [ ] 개인정보처리방침 / 이용약관
- [ ] 검색 기능

## BO 공통 기능 체크리스트

- [ ] 관리자 로그인 / 권한 관리
- [ ] FO 콘텐츠 등록·수정·삭제 (게시판, 배너, 팝업)
- [ ] 회원 목록 조회 (회원 기능 있는 경우)
- [ ] 문의 접수 관리
- [ ] 통계 대시보드 (방문자, 문의 건수 등)
- [ ] 메뉴 구조 관리
