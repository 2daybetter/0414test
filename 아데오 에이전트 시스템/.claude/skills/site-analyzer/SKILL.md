# site-analyzer

## 목적

고객사 URL만 제공된 경우(RFP 없음), WebSearch와 공개 정보로 사이트를 분석하여 유지보수/운영 제안의 근거가 되는 현황 분석서(AY-01)를 생성한다.  
**WebFetch 대신 WebSearch를 사용한다** — SSL 인증서 오류 사이트(innobuildlab.com 등) 처리를 위함.

---

## 입력

| 항목 | 필수 | 비고 |
|------|------|------|
| 고객사 URL | 필수 | 분석 대상 사이트 URL |
| 제안 유형 | 필수 | "유지보수/운영" 고정 |
| 프로젝트명 | 필수 | .status 파일 경로에 사용 |

---

## 처리 절차

### Phase 1: 공개 정보 수집

WebSearch로 다음 키워드 조합을 검색한다:

| 검색 목적 | 검색 키워드 예시 |
|---------|----------------|
| 회사 기본 정보 | `{회사명} 회사소개`, `{회사명} 서비스` |
| 기술스택 | `{도메인} site:linkedin.com`, `{회사명} 개발스택`, `{회사명} 기술블로그` |
| 레퍼런스/포트폴리오 | `{회사명} 포트폴리오`, `{회사명} 고객사례` |
| 사이트 상태 | `{도메인} 속도`, `{도메인} 오류`, `{도메인} 리뷰` |
| 경쟁 환경 | `{회사명} 경쟁사`, `{업종} 시장` |

SSL 오류 또는 직접 접근 불가 시 → WebSearch로만 진행, `(직접 접근 불가 — 공개 정보 기반 분석)` 명시.

### Phase 2: 현황 분석 작성

수집된 정보를 바탕으로 다음 4개 섹션을 작성한다:

#### 섹션 1: 회사 개요

| 항목 | 내용 |
|------|------|
| 회사명 | |
| 업종/사업 분야 | |
| 주요 서비스 | |
| 설립/규모 (추정) | |
| 웹사이트 URL | |

#### 섹션 2: 사이트 현황 분석

| 번호 | 분석 영역 | 현재 상태 | 문제점 | 개선 가능성 |
|------|---------|---------|------|-----------|
| 1 | UX/UI 구조 | | | |
| 2 | 기술스택 (추정) | | | |
| 3 | 성능/속도 | | | |
| 4 | 보안 (SSL 등) | | | |
| 5 | 반응형/모바일 | | | |

#### 섹션 3: 개선 포인트 (최소 5개)

| 번호 | 개선 포인트 | 우선순위 | 유지보수 연관성 |
|------|-----------|---------|--------------|
| 1 | | 상/중/하 | 높음/보통/낮음 |
| ... | | | |

세부 작성 기준: `references/site-analysis-rules.md` 참조.

#### 섹션 4: 기술스택 추정

| 항목 | 추정 내용 | 근거 |
|------|---------|------|
| 프론트엔드 프레임워크 | React / Vue / 기타 | |
| 백엔드 | Node.js / PHP / 기타 | |
| 호스팅 | AWS / 자체서버 / 기타 | |
| CMS | 자체개발 / WordPress / 기타 | |

**미확인 항목**: 확인 불가 정보는 "미확인 (공개 정보 없음)"으로 명시. 임의 추측 금지.

### Phase 3: 검증 및 저장

#### 자기 검증 체크리스트

- [ ] 4개 섹션 (회사개요, 현황분석, 개선포인트, 기술스택) 모두 존재
- [ ] 개선 포인트 5개 이상 작성
- [ ] 각 개선 포인트에 우선순위 및 유지보수 연관성 명시
- [ ] 미확인 항목에 "(미확인)" 표기

검증 스크립트: `scripts/validate_site_analysis.py` 실행

#### 저장 및 URL 기록

1. `gen_requirements.py`로 AY-01 포맷 `.xlsx` 생성  
   → `mcp__claude_ai_Google_Drive__create_file`로 Google Drive `아데오 프로젝트/{프로젝트명}/` 폴더에 `AY-01-현황분석서.xlsx` 업로드  
   → 업로드 후 로컬 `.xlsx` 즉시 삭제
2. 반환된 Google Drive URL을 `.status` 파일의 `outputs.site-analysis`에 기록  
   (`.status` 경로: `아데오 프로젝트/{프로젝트명}/.status/.status`, 루트 폴더 ID: `1XHWdKpQmsoyiScj-NicRrHuYzDZsyBDM`)

## 출력 형식

Google Drive `아데오 프로젝트/{프로젝트명}/AY-01-현황분석서.xlsx`  
`.status` 파일 `outputs.site-analysis` 필드에 Drive URL 기록

## 오류 처리

| 오류 유형 | 처리 방법 |
|---------|---------|
| WebSearch 결과 부족 (3회 이하) | 분석 가능한 항목만 작성 후 "미확인 항목" 섹션 추가, 에스컬레이션 없이 진행 |
| 개선 포인트 5개 미만 | 재검색 1회 후 가능한 항목으로 진행, 보고서에 "(정보 제한)" 명시 |
| Google Drive 업로드 실패 | 오류 내용과 함께 이사님 에스컬레이션 |
| .status 파일 부재 | .status 파일 신규 생성 후 outputs.site-analysis 기록 |
