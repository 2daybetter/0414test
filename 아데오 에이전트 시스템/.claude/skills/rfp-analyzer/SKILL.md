# rfp-analyzer 스킬

## 목적

URL + RFP 파일(또는 텍스트)을 받아 rfp-context 문서를 Google Drive MCP로 업로드한다.  
업로드된 URL이 Google Drive 아데오 프로젝트/{프로젝트명}/.status/.status 파일의 `outputs.rfp-context`에 기록되면 이후 모든 에이전트가 인터뷰 없이 산출물을 자동 생성한다.

---

## 입력

| 항목 | 형식 | 필수 |
|------|------|------|
| 고객사 현재 사이트 URL | URL 문자열 | 필수 |
| RFP 문서 | 파일 경로 또는 텍스트 붙여넣기 | 필수 |

---

## 처리 절차

### Phase 1 — URL 분석 (WebFetch 사용)

URL에 접속하여 다음 정보를 자동 수집한다:

| 수집 항목 | 추출 방법 |
|---------|---------|
| 고객사명 | 페이지 타이틀, 로고 텍스트 |
| 현재 GNB 메뉴 구조 (1~2depth) | 네비게이션 HTML 파싱 |
| 주요 서비스/콘텐츠 카테고리 | 메인 페이지 섹션 분석 |
| 현재 사이트 기술 스택 단서 | meta 태그, 스크립트 소스 |
| 디자인 톤앤매너 | 컬러, 폰트 등 |
| 문제점 / 개선 가능 영역 | 구조·콘텐츠·UX 관점 |

URL 접속 불가 시: "URL 분석 불가 — RFP 기반으로만 진행" 표기 후 계속.

### Phase 2 — RFP 파싱

RFP에서 다음 항목을 추출한다. 항목이 없으면 "미정" 또는 "RFP 미기재"로 표기하고 계속 진행 (중단 금지).

| 추출 항목 | RFP 내 일반적 위치 |
|---------|----------------|
| 사업명 / 프로젝트명 | 표지, 제목 |
| 발주처(기관)명 | 표지, 발주처 정보 |
| 예산 규모 | 사업 개요, 예산 항목 |
| 납기일 / 계약 기간 | 일정, 납기 조항 |
| 담당자 정보 | 문의처, 담당자 섹션 |
| 핵심 기능 요구사항 | 과업 내용, 기능 요구사항 |
| 비기능 요구사항 | 성능, 보안, 반응형 요건 |
| 범위 (포함/제외) | 과업 범위, 제외 항목 |
| 기술 스택 요구사항 | 기술 요건, 개발 환경 |
| 벤치마킹 언급 사이트 | 참고 사이트, 레퍼런스 |
| 평가 기준 | 입찰 평가 기준 (있는 경우) |
| 제출 기한 | 제출 일정, 입찰 기한 |

### Phase 3 — Google Drive 업로드

#### 3-1. 프로젝트 폴더 아이콘 결정

프로젝트명·사업명 키워드를 기준으로 아이콘을 선택하고 폴더명에 붙인다.  
**폴더명 형식**: `{아이콘} {프로젝트명}`

| 우선순위 | 아이콘 | 매칭 키워드 |
|---------|--------|------------|
| 1 | 📱 | 앱, 모바일, iOS, Android, APP, 어플 |
| 2 | ⚙️ | ERP, 시스템, 솔루션, 플랫폼, 인트라넷, 그룹웨어 |
| 3 | 🛒 | 쇼핑몰, 이커머스, 커머스, 상점 |
| 4 | 📊 | 데이터, 분석, AI, ML, 빅데이터, 대시보드 |
| 5 | 🎨 | 디자인, UI, UX, 브랜딩, BI |
| 6 | 💼 | 정책자금, 사업제안서, 입찰, 제안 |
| 7 | 🌐 | 홈페이지, 웹사이트, 포털, 사이트, 개편, 구축, 웹 |
| 기본 | 📁 | (위 키워드 미해당) |

예시: 프로젝트명 "더케이예다함-홈페이지개편" → 키워드 "홈페이지", "개편" → `🌐 더케이예다함-홈페이지개편`  
예시: 프로젝트명 "ABC-모바일앱구축" → 키워드 "모바일", "앱" → `📱 ABC-모바일앱구축`

#### 3-2. 폴더 및 파일 생성

rfp-context 내용을 구성한 후 `mcp__claude_ai_Google_Drive__create_file`로 업로드하고, 반환된 URL을 Google Drive 아데오 프로젝트/{아이콘} {프로젝트명}/.status/.status 파일의 `outputs.rfp-context`에 기록한다.

**폴더 생성 순서**:
1. `mimeType: application/vnd.google-apps.folder`, `title: "{아이콘} {프로젝트명}"`, `parentId: 1XHWdKpQmsoyiScj-NicRrHuYzDZsyBDM` → 프로젝트 폴더 생성 → 반환 ID 저장
2. `mimeType: application/vnd.google-apps.folder`, `title: ".status"`, `parentId: {1의 ID}` → .status 폴더 생성 → 반환 ID 저장
3. rfp-context 파일 업로드 → `parentId: {1의 ID}`
4. `.status` 파일 생성 → `parentId: {2의 ID}`

## 출력 경로

Google Drive MCP(`mcp__claude_ai_Google_Drive__create_file`)로 업로드 → 반환된 URL을 Google Drive 아데오 프로젝트/{아이콘} {프로젝트명}/.status/.status 파일의 `outputs.rfp-context`에 기록.

프로젝트명 추출 규칙: RFP 사업명에서 공백을 `-`로 치환한 단축명. 추출 불가 시 발주처명 사용.  
폴더명 아이콘 규칙: Phase 3-1의 키워드 매핑 표 적용. 폴더 `title`에 아이콘 포함 필수.

---

## 호출 주체

이 스킬은 **영업팀 L3 (sales-team/AGENT.md)** 이 호출한다.  
URL + RFP 동시 제공 시 비서실 L1 → 제안 파트 L2 → 영업팀 L3 순서로 위임되며, 영업팀 L3이 rfp-analyzer를 실행한다. 완료 후 영업팀 L3이 제안 파트 L2에 보고하고, 제안 파트 L2 → 구축 파트 L2 순서로 이후 단계가 진행된다.  
제안 파트 L2 및 각 L3 에이전트는 rfp-context.md가 이미 생성된 것으로 가정한다.

## 오류 처리

| 상황 | 처리 방법 |
|------|---------|
| URL 접속 불가 | "URL 분석 불가" 표기 후 RFP만으로 진행 |
| RFP에서 예산 미확인 | "RFP 미기재" 표기 후 계속 진행 |
| RFP에서 납기 미확인 | "미정 — 착수 후 합의" 표기 후 계속 진행 |
| RFP에서 담당자 미확인 | "RFP 미기재" 표기 후 계속 진행 |
| 기능 요구사항 5개 미만 추출 | "RFP 내용 부족 — 추출된 항목만으로 진행" 표기 |

**어떤 경우에도 rfp-context.md 생성을 중단하지 않는다.**  
모든 미확인 항목은 "미정" 또는 "RFP 미기재"로 표기하고 파일을 완성한다.
