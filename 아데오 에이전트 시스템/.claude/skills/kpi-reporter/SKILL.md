# kpi-reporter

## 개요

`/output/` 디렉토리 스캔 결과를 기반으로 주간(PM-WR) 또는 월간(PM-MR) KPI 보고서를 생성한다.  
반드시 `/templates/weekly-report-template.md` 또는 `/templates/monthly-report-template.md` 를 참조하여 작성한다.  
세부 집계 기준은 `references/kpi-rules.md` 참조.

---

## 실행 절차

### Step 1: 보고 유형 결정

| 조건 | 보고 유형 | 출력 경로 |
|------|---------|---------|
| 주간 보고 요청 또는 매주 월요일 트리거 | 주간 (PM-WR) | `/output/비서실/weekly-{YYYYMMDD}.md` |
| 월간 보고 요청 또는 매월 1일 트리거 | 월간 (PM-MR) | `/output/비서실/monthly-{YYYYMM}.md` |

날짜 미제공 시 현재 날짜(오늘) 기준으로 산정.

### Step 2: 데이터 수집

`scripts/collect-kpi.py` 스크립트를 실행하여 `/output/` 디렉토리를 스캔한다.

```bash
python -X utf8 scripts/collect-kpi.py
```

스크립트가 없는 경우 다음 항목을 수동 탐색으로 수집한다:

| 수집 항목 | 탐색 경로 | 집계 방법 |
|---------|---------|---------|
| 구축 파트 진행 프로젝트 수 | `/output/구축 파트/` | 하위 디렉토리(프로젝트명) 수 집계 |
| 완성 산출물 수 | `/output/구축 파트/**/*.md` | 파일 수 집계 |
| 현재 단계 분포 | `/output/구축 파트/**/.status` | `current_step` 필드 집계 |
| 제안 파트 활동 수 | `/output/제안 파트/` | 하위 디렉토리(기회명) 수 집계 |
| 제안서 완성 수 | `/output/제안 파트/**/전략팀/제안서-*.md` | 파일 수 집계 |
| 정책자금 매칭 수 | `/output/제안 파트/**/정책자금팀/*.md` | 파일 수 집계 |

### Step 3: KPI 지표 산출

세부 지표 정의 및 목표값은 `references/kpi-rules.md` 참조.

| KPI 지표 | 계산 방법 |
|---------|---------|
| 산출물 생성 속도 | 해당 기간 신규 파일 수 / 영업일 수 |
| 단계별 진행 현황 | .status 파일의 current_step 분포 |
| 제안 활동 현황 | 신규 기회 수 / 제안서 완성 수 / 수주 여부 |
| 정책자금 매칭률 | 매칭 완료 수 / 전체 조회 요청 수 |
| 미완료 항목 | .status에서 locked_by가 7일 이상 변경 없는 항목 |

### Step 4: 보고서 작성

- **주간**: 지난 7일간 활동 요약 + 이번 주 계획 + 미완료 항목
- **월간**: 지난 월간 주요 성과 + KPI 달성률 + 다음 월 계획

LLM이 수치 데이터를 기반으로 서술형 해석(1~2문단)을 추가한다.  
해석 작성 시 확인되지 않은 수치나 추정값을 사실처럼 기술하지 말 것.

### Step 5: 출력 및 저장

- **주간 경로**: `/output/비서실/weekly-{YYYYMMDD}.md`
- **월간 경로**: `/output/비서실/monthly-{YYYYMM}.md`
- **성공 기준**: KPI 지표 섹션 존재 + 활동 요약 + 미완료 항목 목록 포함

## 실패 처리

| 실패 유형 | 처리 |
|---------|------|
| `/output/` 디렉토리 비어있음 | "집계 대상 없음" 보고서 생성 후 정상 완료 |
| collect-kpi.py 실행 실패 | 수동 탐색으로 대체 후 비고에 "스크립트 미실행" 표시 |
| 날짜 정보 없음 | 오늘 날짜 기준으로 자동 산정 |
