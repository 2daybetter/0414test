# project-kickoff

## 개요

제안사 인계 문서 기반으로 아데오 표준 착수 문서 패키지(PM-01~03)를 생성한다.  
반드시 `/templates/kickoff-template.md` 를 참조하여 작성한다.

---

## 실행 절차

### Step 1: 입력 확인

| 필수 입력 | 확인 방법 |
|---------|---------|
| 프로젝트명 | 사용자 입력 또는 인계 문서에서 추출 |
| 납기일 | 인계 문서 또는 사용자 입력 (YYYY-MM-DD) |
| 고객사명 | 인계 문서에서 추출 |
| 계약 금액 (선택) | 있으면 반영 |
| 킥오프 템플릿 | `/templates/kickoff-template.md` 존재 여부 |

납기일이 없으면 **[확인 요청]** 을 출력하고 중단.

### Step 2: 마일스톤 일정 계산

납기일 기준 역산하여 6단계 시작/종료일 초안 계산.  
세부 일정 계산 규칙은 `references/kickoff-rules.md` 참조.

> ⚠️ 계산 후 반드시 **[확인 요청]** 으로 사람 승인을 받는다. 승인 전 WBS 작성 금지.

**[확인 요청] 출력 형식**:
```
[확인 요청] WBS 납기일 및 마일스톤 확인
─────────────────────────────────────
프로젝트: {프로젝트명}  고객사: {고객사명}
최종 납기일: {YYYY-MM-DD}

단계별 일정 (안):
  PM (착수, 5%)   : {시작} ~ {종료}
  AY (분석, 10%)  : {시작} ~ {종료}
  DE (설계, 20%)  : {시작} ~ {종료}
  IM (구현, 45%)  : {시작} ~ {종료}
  TE (테스트, 10%): {시작} ~ {종료}
  OP (오픈, 10%)  : {시작} ~ {종료}

위 일정으로 진행하겠습니다. 수정이 필요하면 알려주세요.
승인 시 "확인" 또는 수정 내용을 입력해주세요.
─────────────────────────────────────
```

### Step 3: WBS 작성 (PM-03)

승인된 일정 기준으로 `/templates/kickoff-template.md` 를 채운다.

| 작성 항목 | 내용 |
|---------|------|
| 6단계 마일스톤 테이블 | 단계 / 코드 / 가중치 / 시작일 / 종료일 / 담당팀 |
| 단계별 산출물 코드 배분 | PM-01~03, AY-01, DE-02~08, IM-01~04, TE-01~08, OP-01~05 |
| 팀별 담당 업무 | PM팀 / 웹기획팀 / 디자인팀 / 개발팀별 단계별 역할 |
| 리스크 목록 초안 | 일정 지연 리스크, 요구사항 변경 리스크, 기술 리스크 최소 3개 |
| 커뮤니케이션 계획 | 주간 보고 일정, 이사님 보고 기준 |

### Step 4: 착수보고서 작성 (PM-01)

| 항목 | 내용 |
|------|------|
| 프로젝트 개요 | 목적, 배경, 범위 |
| 프로젝트 조직 | PM, 팀별 담당자 |
| 마일스톤 요약 | 6단계 일정 요약 |
| 산출물 계획 | 문서 코드별 생성 계획 |
| 승인 요청 | 이사님 착수 승인 항목 |

### Step 5: 인력계획서 작성 (PM-02)

| 항목 | 내용 |
|------|------|
| 팀 구성 | PM / 웹기획 / 디자인 / 개발팀 |
| 단계별 투입 인원 | 각 팀의 단계별 투입 인력 수 |
| 역할 정의 | 팀별 담당 역할 및 책임 |

### Step 6: 출력 및 검증

- **출력 경로**: `/output/구축사/{프로젝트명}/PM/kickoff-{프로젝트명}.md`
- **검증**: `scripts/validate-doc.py` 실행
- **성공 기준**: PM/AY/DE/IM/TE/OP 6단계 섹션 + PM-01~03 산출물 코드 + 납기일 반영

## 실패 처리

| 실패 유형 | 처리 |
|---------|------|
| 납기일 없음 | [확인 요청] 출력 후 중단 |
| 일정 역산 불가 (납기 너무 촉박) | 최소 일정 경고 후 사람 판단 요청 |
| validate-doc.py 실패 | 자동 재시도 1회 → 초과 시 에스컬레이션 |
