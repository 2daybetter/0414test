##### 1.3.2 개발 환경 구성 방안 (요구사항 ID: ECR-003)

> React와 Spring Boot 3.x 등 최신 기술을 적용해 최적의 사용자 경험을 제공하고, 유지보수와 재사용이 편리한 효율적 개발 환경을 구축합니다. 자동화된 CI/CD 파이프라인 구성과 빠른 장애 대응을 위한 Rolling 롤백 전략으로 안정성을 강화합니다.

| 구분 | 기술 스택 | 적용 내용 |
|------|---------|---------|
| 백엔드 | Spring Boot 3.x (Java 17 이상) | RESTful API 서버 설계, Spring Security (인증/인가), Hibernate (DB 연동), Lombok/MapStruct |
| 프론트엔드 | React 18 이상 | 상태 관리: React Query / 라우팅: React Router / UI 라이브러리: Material-UI / 반응형 디자인 |
| 소스 코드 관리 | GitLab | GitLab 리포지토리 연동, 브랜치 전략 적용 |
| 배포 자동화 | GitLab CI/CD | 소스 코드 변경 시 자동 빌드 및 배포, GitLab Runner 활용 |
| 배포 안정성 | Rolling 배포 | 서버 그룹 중 일부씩 순차적으로 새 버전으로 교체, 서비스 중단 없는 배포 |
| 배포 정확성 | CI/CD Pipeline | 단계별 로그 및 실패 처리 로직, 배포 자동화 도구로 빌드/배포 과정 오류 최소화 |

##### 1.3.3 성능 최적화 방안 (요구사항 ID: PER-001, PER-002)

> JMeter를 활용해 성능·부하 테스트를 실시하고, 시스템 로그와 사용량을 분석하여 각 시스템의 성능 적합성을 검증합니다. 또한 기존 시스템과의 호환성과 동시처리 성능을 보장하도록 안정적으로 구성합니다.

**성능 검증 절차**: Detect(감지) → Find(발견) → Focus(집중) → Improve(향상) → Verify(검증)

| 성능 진단 대상 | 진단 내용 |
|-------------|---------|
| 현황 분석 | 동시 사용자수가 최대인 시점으로 측정 / 1 사용자당 초당 TPS |
| 부하 분석 | 기준 사용자 000명 기준 CPU 부하 측정 |
| 처리량 예측 | 현재 TPS 대비 처리 가능한 최대 사용자 수 예측 |

- JMeter를 통해 가상 유저로 부하를 생성하여 각 구간별 처리시간을 측정
- 가상 유저는 성능 검증 기준의 Active User를 기준으로 하되 50%, 200%의 가상 유저로 테스트하여 시스템 안정성 검증
- 구간별 측정 결과를 분석하여 병목구간 및 성능 저하 요소를 판별

**성능 모니터링 영역**: 네트워크 / WEB 서버 / WAS 서버 / 데이터베이스 / 연계시스템

##### 1.4.1 데이터 표준 수립 및 운영방안 (요구사항 ID: DAR-001, DAR-002, DAR-004)

> 표준화 절차를 통해 데이터 표준을 수립하고, 일관성·종속성·무결성 및 DB 성능을 보장하는 구조로 설계합니다. 또한 개인정보와 민감정보는 고객사 내규 및 관련 법령·지침에 따라 암호화해 안전하고 적법하게 처리합니다.

**표준화 절차**

| 단계 | 내용 |
|------|------|
| 업무현황 조사 → 업무요건 정의 | 현 시스템 데이터 조사 |
| 업무 참조모델 제시 → 데이터 표준개발 | 표준화 준수 운영체계 수립 |
| 현업 요구사항 분석 → 반영 | 차기 프로젝트 반영 |
| | 메타데이터 관리 테이블 적용 → 운영 관리 |

**데이터 표준화 프로세스**

일정 협의 → 표준 항목 개발 (명명규칙 / 표준용어 / 코드부여규칙 / 표준코드 / 공통코드 통합 / 도메인표준) → 논리 모델링 적용 (주제영역 / 엔티티타입 / 어트리뷰트) → 물리모델링 적용 (테이블 / 컬럼) → 데이터베이스 생성 (Character Set: **UTF-8**)

**통합코드 표준화 작업**

| 단계 | 내용 |
|------|------|
| 코드 준비 | 통합 관리 지침수립 → 통합 코드 운영사례 취합 |
| 채번 방식 취합 | 시스템별 번호 수집 → 표준화 적용 속성 선정 |
| 코드 분석 작업 | 채번 기준 분석 → 통합 코드 생성규칙 작성 |

##### 1.4.2 데이터 이행 및 목표시스템 데이터 검증 (요구사항 ID: DAR-003)

> 목표 시스템은 기존 시스템 이행이 가능하도록 설계하고, 법령·규정·매뉴얼을 준수한 업무규칙(BR)을 정의해 구축합니다. 데이터 이관 후 진단 계획 수립, 진단 수행 및 검증, 사전 오류 방지 순서로 철저히 데이터 검증을 진행합니다.

**데이터 이행 프로세스**: 추출(As-Is 운영서버) → 전송(ftp) → 매핑 (공통모듈 File / 통합 매핑 File) → 적재 (테이블 적재 Loader) → 이관 시 고려사항 (초기 데이터 이관 → 변경분 이관 → 실 이관)

**데이터 검증 절차**

1. **데이터 진단 계획 수립**: 검증 범위 – 전체 데이터 / 검증 시기 – 구축 중반, 종반 / 검증 방법 – 전수 조사, 자동화 스크립트
2. **데이터 진단 수행 및 검증**: 대상 데이터 분석 / 근거 문서 확인 / 검증결과 처리
3. **사전 오류 방지 방안 (입력단계)**: 사전 검증 로직 도입

### 2. 보안관리

#### 2.1 보안 규정 준수 방안 (요구사항 ID: SER-001, SER-002, SER-003)

> 제안사는 도입 소프트웨어 및 사업 수행 중 접근하는 데이터, 시스템, 인력에 대한 보안 정책을 수립하고, 프로젝트 수행 기간은 물론 사업 완료 후에도 엄격한 비밀 보안 조치를 준수하여 안전한 사업 수행을 보장합니다.

**외부 보안기준 및 개인정보영향평가 반영**

| 기관 | 기준 | 내용 |
|------|------|------|
| 국가정보원 | 정보보안 관리실태 평가기준 (9개 부문, 246개 항목) | 국가 보안 기준 준수 |
| 한국인터넷진흥원 | 주요정보통신 기반시설 기술적 취약점 분석평가 상세 가이드 (8개 부문, 62개 항목) | 취약점 분석 및 보완 |

**도입 대상 소프트웨어에 대한 보안 조치 방안**

- 검수 전 시스템에 대한 보안 설정목, 취약점 점검항목에 대한 점검 결과를 제출
- 소프트웨어보안취약점 발견 시 즉시 개선조치 및 결과보고서 제출 / OS 및 응용프로그램의 보안 패치 최신화
- 공개 S/W의 보안취약점 공개 시 업데이트 / Telnet, FTP 등 불필요한 서비스와 포트 제거
- 서버, 네트워크 등 모든 장비는 초기 설치 시 발생할 수 있는 다양한 보안취약점을 사전 분석하고 보안취약점을 제거한 안전한 제품으로 구성
- 정기적인 보안교육 실시 (분기별 1회)
- 사업착수 시 보안서약서, 사업진행 시 보안교육실시 및 내역서, 사업종료 시 모든 자료 반납 하고 삭제 및 대표자명의 보안확약서제출

**보안취약점 진단 절차**: 보안취약점 진단 계획 → 점검기준 검토 → 진단 대상 선정 → 보안취약점 진단 및 조치

| 점검기준 | 진단 대상 | 보안취약점 진단 및 조치 |
|--------|---------|-------------------|
| 웹 취약점 점검 / 보안사고 시 영향도 분석 / 보안대책 수립 | 정통신기반 보호법 / OWASP TOP 10 / 국정원 홈페이지 8대 취약점 | 통합 보안관리 체계 고려 / 보안성 점검 체계 협의 / 담당자와 대상 및 범위 협의 |

**시스템, 네트워크, 소프트웨어, 데이터베이스 보안 조치 방안**

- 디폴트 계정 삭제 후 관리계정을 별도 생성하여 운영 / 애플리케이션 패스워드는 보안규정에 맞게 설정 (디폴트 패스워드 사용금지)
- 국가정보보안기본지침에의거 패스워드 등의 암호화대상 정보 일방향 암호화 알고리즘(SHA512이상) 적용
- 정보시스템접속 및 정책변경 등 모든 접근기록이저장되도록 설계 및 개발
- Batch Job을 통한 시스템 로그 자동백업 및 보관 정책 수립 (최소 2년 이상)
- I/F시 SSL 인증서를 통한 구간암호화 통신을 개발

#### 2.2 모의해킹 수행 방안 (요구사항 ID: SER-004)

> 모의해킹은 사전 협의한 대상과 범위에서 정보 수집과 위협 분석을 선행해 심층적인 취약점 분석을 수행합니다. 도출된 취약점은 근본적 조치가 가능하도록 최적의 개선방안을 제시하며, 소스코드 진단은 자동화 솔루션을 활용합니다.

**모의해킹 절차**

사전 협의 → 정보 수집 → 위협 목록화 → 취약점 점검 및 침투 → 결과 보고

| 단계 | 내용 |
|------|------|
| 사전 협의 | 요구사항 수립 / 대상·범위 선정 / 수행계획 확정 |
| 정보 수집 | 대상 정보수집 (공개출처정보, 인프라, 애플리케이션) / 서비스·업무 파악 |
| 위협 목록화 | 잠재적 취약성 식별 / 공격자료 수집·분석 / 위협 구체화 |
| 취약점 점검 및 침투 | 취약점 검사 / 취약점 연계 공격 / 거점기반 추가 공격 |
| 결과 보고 | 대응방안 마련 / 보고서 제출 / 이행조치 지원 |

**취약점 점검 방안**

- 제안 요구사항에 따른 체크리스트 기반 취약점 점검과 서비스·업무 맞춤형 위협 기반 취약점 점검 병행
- KISA 취약점 분석·평가항목 (주요정보통신기반시설기준) 기준 점검
- 웹 소스코드 자동진단 솔루션 활용하여 점검

| 취약점 점검 항목 | 내용 |
|-------------|------|
| 서비스·업무에 대한 위협 식별 및 목록화 | Apache Log4j 등 최신 취약점 포함 |
| 공개된 취약점·공격코드와 최근 보안동향 반영 | CVE 데이터베이스 활용 |
| 발견한 취약점을 악용하여 추가 연계 공격 | 침투 테스트 수행 |
| 공격거점 확보 후 내부 이동 및 중요정보 탈취 시나리오 | 실전 공격 시뮬레이션 |

### 1. 교육 및 기술지원

#### 1.1 교육 지원 (요구사항 ID: PSR-001)

> 제안사는 체계적이고 효율적인 교육을 위하여 교육훈련에 대한 계획서를 작성하여 제출하고 업무별 / 시스템별 / 교육 대상별 교육자료를 제작 및 배포하여 운영 관리자, 현업부서, 당사 내부 유지보수 인력을 대상으로 교육을 실시합니다.

**시스템별, 대상별, 사업추진 단계별 교육계획서에 따른 관련 대상자 맞춤 교육 진행**  
**체계적이고 효율적인 대상자별 맞춤 교육 진행**

| 구분 | 역할 |
|------|------|
| [발주사] 교육훈련 담당자 | 교육대상, 일정 등 교육훈련 계획 협의 및 승인 / 교육계획에 따른 교육대상자 선정 및 소집 / 운영자 교육 수강 |
| 교육훈련 총괄 책임자 (제안사) | 교육훈련 계획 수립 및 교육훈련 담당자와 협의 / 교육훈련 집행 및 관리 / 수행사 교육훈련 및 기술이전 교육 / 교육 & 기술이전 계획 및 지원계획 수립 |
| 수행사 교육 훈련팀 | 사용자/운영자 교육자료 및 매뉴얼 작성 및 배포 / 1차 교육 (M+5): 시스템 운영자 및 사용자 매뉴얼 교육 실시 / 2차 교육 (M+6): 시스템 구성 및 운영방법, 장애 예방 및 대응방법 교육 / 3차 교육 (M+7): 핵심 개발 상주인력 맞춤 교육 진행 |
| 교육 지원팀 | 상세 교육훈련 계획 수립 및 실적 보고 / 교육훈련 환경 지원 (교재, 교육장 및 기타 환경 준비 등) |

**교육훈련 계획 절차**: 분석 진단 → 교육과정 선정 → 대상자 선정 → 교육 실시 → 사후 관리

- **교육훈련 계획서 제출**: 시스템별, 대상별, 사업추진 단계별 교육훈련 방안을 제시하여야 함 (승인 필요) / 교육계획서는 교육훈련 목적, 내용, 대상, 방법, 일정, 장소 등에 대한 상세사항을 포함
- **교육자료 제작**: 운영자 및 사용자 매뉴얼을 화면별·단계별·절차별로 상세하게 작성 / 화면 및 기능별 오류 메시지, 주석 및 로그를 통한 오류 상황을 정확히 파악하여 대응할 수 있도록 교육자료 제작
- **교육대상을 고려한 교육**: APP, P/W, M/W 관리, 운영과 관련된 관리자(콘텐츠, 시스템), 현업부서 사용자, 당사 내부 유지보수 인력 등 교육 대상자를 차별화하여 교육 실시
- **도입 솔루션 교육**: APP, P/W, M/W 관리 및 운영과 관련된 관리자(콘텐츠, 시스템 및 제반 상용 솔루션)와 당사 유지보수 인력을 대상으로 교육 실시

### 2. 유지관리

> 제안사는 H/W 및 S/W 장애에 대한 대책 수립 및 자료의 파손, 변질, 분실 등에 대비하기 위하여 체계적이고 효과적인 최단시간 내 실행 가능한 백업 및 복구방안 계획을 수립하고 무상 하자보수 기간 내 해당 업무를 수행합니다.
