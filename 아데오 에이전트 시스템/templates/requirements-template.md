## 2. 기능 요구사항

> **시트명**: `기능`  
> **출력 형식**: `scripts/generators/gen_requirements.py`로 `.xlsx` 생성 → Google Drive MCP 업로드

### 기능 탭 컬럼 구조

| 컬럼 | 설명 |
|------|------|
| 요구사항 ID | FUR001부터 F/O·B/O 통합 순번 순차 부여 |
| 시스템 | `F/O` (Front Office) 또는 `B/O` (Back Office) |
| 구분 | 1depth 메뉴 또는 기능 카테고리 |
| 요구사항명 | 기능의 명칭 (짧고 명확하게) |
| 요구사항 설명 | 세부 기능 설명 (bullet 형식 권장) |
| 출처 | 요구사항 도출 근거 (예: "RFP 및 고객사 전달 문서, 인터뷰") |
| 변경내용 | 기존 요구사항 대비 변경사항 (최초 작성 시 `-`) |
| 관련근거 | 관련 회의록, 화면설계서 페이지 등 문서 근거 |
| 수용여부 | 수용 / 미수용 / 부분수용 / 삭제 |
| 비고 | 특이사항, 우선순위 메모, 연관 기능 등 |

### 수용여부 값 정의

| 값 | 의미 |
|----|------|
| 수용 | 요구사항 전체 반영 |
| 미수용 | 요구사항 반영 불가 (사유 비고란 기재) |
| 부분수용 | 일부만 반영 (내용 비고란 기재) |
| 삭제 | 요구사항 자체 제거 (중복·범위 외) |
