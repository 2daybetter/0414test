# Spec: 예다함 홈페이지 운영·유지보수 및 온라인 광고 통합 운영

## Overview
더케이예다함(yedaham.co.kr) 기 구축 시스템(2025-12-18 오픈)의 10개월 운영·유지보수 및 온라인 광고 통합 운영 사업. 계약기간 2026.03~2026.12.31, 총 사업비 427,920,000원. React19 + SpringBoot3.5.3 + Kotlin2.2.0 + Oracle 11g R2 기반 운영 체계 수립 및 서비스 안정성 확보가 핵심 목표.

## Goals
- 24/365 무중단 서비스 운영 (장애 인지 4시간 내 조치, 24시간 내 복구)
- 기존 운영자 시스템(CMS) 운영 및 콘텐츠 관리 대행
- 온라인 광고(네이버·구글·SNS·유튜브) 통합 운영 및 성과 분석
- 결함 수정·기능 개선·보안 패치 이행 (MPR 기준 준수)
- GA4 기반 월별 방문자 통계·광고 성과 보고서 제공

## Scope
- **In**:
  - 홈페이지(yedaham.co.kr) 운영·유지보수 (React19 프론트엔드, SpringBoot3.5.3/Kotlin2.2.0 백엔드)
  - 모바일 앱(React Native) 운영·유지보수
  - 운영자 시스템(CMS) 콘텐츠 관리: 홈페이지관리, 회사사이트관리, 법인사이트관리, 제휴사관리, 통계관리, 이력관리, 시스템관리
  - 온라인 광고 통합 운영: 네이버 검색광고, 구글 광고, 브랜드 검색, 배너광고, 유튜브, SNS 바이럴
  - GA4 기반 성과 보고 (월별 리포트)
  - 보안 취약점 점검 (연 1회 이상) 및 패치 적용
  - 웹표준·접근성(한국형 웹 콘텐츠 접근성 지침 2.1) 준수 유지
- **Out**:
  - 신규 시스템 개발 (기 구축 시스템 운영만 포함)
  - 실제 광고비 집행 (별도 예산 ~83억원, 계약 외)
  - 물리 인프라 구축 (KT Cloud CDN 기 운영 중)
  - 회원가입·결제 시스템 신규 개발

## Inputs and Outputs
- **입력**: 기 구축 시스템 소스코드·DB(Oracle 11g R2), 운영자 시스템 접근 권한, GA4 데이터, 광고 계정 접근 권한
- **출력**: 월별 운영 보고서(GA4 통계·광고 성과·결함 처리 현황), 장애 처리 보고서, 보안 점검 결과 보고서, 서비스 개선 결과물
- **트리거**: 클라이언트 요청 / 모니터링 경보 / 월별 정기 보고 일정

## Requirements

### Functional

#### MPR: 유지관리 수행 요구사항 (9건)

- **MPR-001 CMS 운영**: 운영자 시스템 7개 모듈(홈페이지관리·회사사이트관리·법인사이트관리·제휴사관리·통계관리·이력관리·시스템관리) 콘텐츠 관리 대행. 비개발자 요청 처리 포함
- **MPR-002 결함 수정**: 기능 오류·화면 깨짐·데이터 오류 발생 시 원인 분석 및 수정 코드 배포
- **MPR-003 서비스 모니터링**: 24/365 서버·DB·네트워크 모니터링. 임계값 초과 시 즉시 알림
- **MPR-004 장애 조치**: 장애 인지 4시간 내 조치 착수, 24시간 내 서비스 복구. 장애 보고서 제출
- **MPR-005 웹표준 준수**: 한국형 웹 콘텐츠 접근성 지침 2.1 준수. 개선 요구 발생 시 수정
- **MPR-006 성능 최적화**: 응답 속도 3초 이내 유지 (PER-001 연계). 느린 쿼리·프론트 번들 최적화
- **MPR-007 백업 및 복구**: DB·파일 정기 백업 계획 수립 및 이행. 복구 테스트 분기 1회
- **MPR-008 콘텐츠 갱신**: 상품 정보(윤슬4·윤슬6·노블), 혜택 서비스, 공지사항 등 운영 콘텐츠 업데이트
- **MPR-009 신규 기능 반영**: 경미한 기능 추가·변경 요청 처리 (공수 5M/M 이내 월별 조율)

#### MHR: 투입 인력 요구사항 (3건)

- **MHR-001 PM**: 경력 10년 이상, 유사 프로젝트 관리 경험. 사업 전체 일정·품질·의사소통 총괄
- **MHR-002 디자인·기획**: 경력 8년 이상. UI/UX 개선·콘텐츠 제작·광고 소재 디자인 담당
- **MHR-003 개발 (프론트·백엔드)**: 프론트엔드 6년 이상(React19), 백엔드 10년 이상(SpringBoot/Kotlin/Oracle). 비상주 투입 기준

총 투입 인력: 4.5M/M (PM/기획 1.0, 디자인 1.5, 프론트엔드 0.5, 백엔드 1.5)

#### PER: 성능 요구사항 (1건)

- **PER-001**: 응답 시간 3초 이내, 동시 접속 100명 처리. 성능 저하 감지 시 원인 분석 및 조치

#### DAR: 데이터 요구사항 (4건)

- **DAR-001 데이터 표준**: UTF-8 인코딩 유지. 신규 데이터 항목 추가 시 표준 준수
- **DAR-002 데이터 구조**: Oracle 11g R2 기반 기존 스키마 변경 시 변경 이력 관리
- **DAR-003 데이터 검증**: 입력값 유효성 검사 유지. 개인정보 항목 SHA512+ 암호화 확인
- **DAR-004 데이터 설계**: 신규 기능 추가 시 ERD 문서화, DBA 승인 후 배포

#### SER: 보안 요구사항 (3건)

- **SER-001 정보보안**: OWASP Top 10 취약점 점검. 보안 취약점 발견 시 48시간 내 패치
- **SER-002 인프라보안**: KT Cloud 인프라 보안 설정 유지·검토. SSL 인증서 만료 관리
- **SER-003 인적보안**: 접근 권한 관리(최소 권한 원칙), 퇴직자 계정 즉시 비활성화. 연 1회 보안 취약점 점검 수행

#### COR: 준수 요구사항 (3건)

- **COR-001 업무표준**: 코드 리뷰·배포 절차·이슈 트래킹 표준 수립 및 준수
- **COR-002 정보화표준화지침**: 행정안전부 정보화표준화지침 준수 여부 정기 확인
- **COR-003 저작권**: 사용 이미지·폰트·라이브러리 라이선스 관리. 무단 사용 금지

#### PSR: 유지보수 방안 요구사항 (1건)

- **PSR-001**: 월별 정기 유지보수 계획서 제출. 결함 처리 현황·성능 지표·보안 이슈 포함. 분기별 서비스 개선 계획 수립

#### PMR: 사업관리 요구사항 (4건)

- **PMR-001 사업관리**: 킥오프 미팅, 주간 보고, 월별 결과 보고서 제출
- **PMR-002 인력관리**: 투입 인력 변경 시 30일 전 사전 통보·승인. 대체 인력 동급 이상 유지
- **PMR-003 요구사항관리**: 클라이언트 요청 사항 접수 → 우선순위 분류 → 처리 일정 협의 → 완료 확인 프로세스 운영
- **PMR-004 위험관리**: 위험 요소 식별·평가·대응 계획 월별 갱신. 주요 위험: 핵심 인력 이탈, 인프라 장애, 보안 사고

#### 온라인 광고 운영

- 광고 채널: 네이버 검색광고, 구글 광고, 브랜드 검색, 배너광고, 유튜브, SNS 바이럴
- 타깃 분석: GA4 데이터 기반 방문자 세그먼트 분석, 전환율 개선 방향 도출
- DB 기반 마케팅: 기 보유 고객 데이터 활용 타깃 광고 집행 (개인정보보호법 준수)
- SNS 콘텐츠: 월별 콘텐츠 캘린더 수립, 장례서비스 특성에 맞는 신뢰 중심 콘텐츠 제작
- 성과 보고: GA4 기반 채널별 유입·전환·비용 효율 월별 리포트

### Non-functional

- 서비스 가용성: 월 99.5% 이상 (계획된 점검 시간 제외)
- 응답 시간: 3초 이내 (PER-001)
- 암호화: 개인정보 SHA512+ 암호화, HTTPS 강제 적용
- 접근성: 한국형 웹 콘텐츠 접근성 지침 2.1 AA 수준 유지
- 웹표준: W3C 표준 준수
- 보안: OWASP Top 10 대응, 연 1회 이상 취약점 점검

## Technical Notes

- **기존 스택**: React19 (프론트엔드) + SpringBoot3.5.3 + Kotlin2.2.0 + JPA3.5.1 + JDK21 (백엔드)
- **DB**: Oracle 11g R2
- **Web/WAS**: Nginx + Tomcat Embed Core
- **인프라**: KT Cloud CDN
- **모바일**: React Native
- **본인인증**: NICE·라온시큐어 연동 (기 구축)
- **인증**: SHA512+ 암호화 (기 적용)
- **기존 메뉴 구조**: MY, 상품안내(윤슬4/윤슬6/노블), 간편가입, 혜택서비스, 회사소개(corp), 법인소개(ltd)
- **CMS 모듈**: 홈페이지관리, 회사사이트관리, 법인사이트관리, 제휴사관리, 통계관리, 이력관리, 시스템관리
- **애널리틱스**: GA4 (`gtag.js`) 기 연동, Reporting API로 월별 리포트 자동화
- **배포 방식**: 현행 배포 파이프라인 인수인계 후 유지 (CI/CD 파악 후 문서화)

## UI/UX Notes

- 기 구축된 디자인 시스템 유지 (변경 최소화 원칙)
- UI 개선 요청 발생 시 기존 디자인 컨셉(차분·전문적·신뢰감) 유지
- 접근성 개선: 시니어 사용자 고려, 최소 폰트 16px, 클릭 영역 최소 44px 기준 준수
- 모바일 최적화: 기존 React Native 앱과 웹 일관성 유지

## Tradeoffs & Decisions

| 결정 | 선택 | 포기한 것 | 이유 |
|------|------|----------|------|
| 개발 투입 방식 | 비상주 (필요 시 현장 방문) | 상주 | RFP 비상주 명시, 비용 효율 |
| 광고 플랫폼 | 네이버·구글·SNS 멀티채널 | 단일 채널 집중 | RFP 다채널 명시, 타깃 커버리지 |
| CMS | 기 구축 운영자시스템 운영 | 신규 CMS 도입 | 기 구축 자산 활용, 전환 비용 없음 |
| 모니터링 | 24/365 자동 경보 체계 | 업무시간만 모니터링 | 장례서비스 특성상 24시간 긴급 대응 필요 |
| 보고 주기 | 월별 정기 + 장애 즉시 | 분기 단위만 | PMR-001 월별 보고 요구사항 |

## Failure Modes

- **인프라 장애**: KT Cloud 의존 → 장애 등급 정의 및 에스컬레이션 절차 수립, 4h 조치/24h 복구 SLA 준수
- **핵심 인력 이탈**: 대체 인력 Pool 사전 확보, 인수인계 문서화 상시 최신화
- **보안 침해**: OWASP 대응 프로세스, 침해 시 즉시 서비스 격리 및 보고 절차
- **광고 계정 접근 불가**: 멀티 계정 관리자 등록, 접근권한 이중화
- **DB 데이터 손실**: 일일 백업 + 분기 복구 테스트, Oracle 아카이브 로그 설정 확인
- **CMS 권한 오류**: 변경 전 스테이징 환경 검증, 권한별 접근 로그 기록

## Open Questions

- 배포 파이프라인 현황 (CI/CD 도구·서버 접속 방식) 인수인계 전 파악 필요
- 월별 광고 집행 예산 승인 절차 및 담당자 확인 필요
- Oracle 11g R2 라이선스 유효기간·지원 정책 확인 (EOL 리스크)
- 기존 코드 레포지토리 접근 권한 및 브랜치 전략 인수인계 필요
- 분기 1회 복구 테스트 시 서비스 중단 허용 여부 협의 필요
- SNS 운영 채널(인스타그램·유튜브 등) 현황 및 계정 접근 권한 확인
