# 더케이예다함 홈페이지 운영·유지보수 및 온라인 광고 통합 운영 제안서

> 제안일: 2026-04-20
> 사업기간: 2026.03.01 ~ 2026.12.31 (10개월)
> 총 사업비: 427,920,000원 (VAT 포함)

---

## 1. 제안 개요

### 1.1 사업 목적

더케이예다함(yedaham.co.kr)은 2025년 12월 리뉴얼 오픈한 디지털 서비스 채널을 안정적으로 운영하고, 온라인 광고를 통한 가입 전환을 극대화하기 위해 전문 운영 파트너를 필요로 합니다.

본 제안사는 다음 세 가지 핵심 가치를 제공합니다:

- **무중단 운영**: 24/365 모니터링 및 SLA 기반 장애 대응 (4시간 내 조치, 24시간 내 복구)
- **콘텐츠·서비스 안정화**: CMS 운영 대행 및 결함 수정을 통한 서비스 품질 유지
- **광고 성과 최적화**: GA4 기반 데이터 분석으로 채널별 ROI 극대화

### 1.2 제안 범위 요약

| 구분 | 내용 |
|------|------|
| 운영·유지보수 | 홈페이지(React19/SpringBoot/Oracle) + 모바일 앱(React Native) |
| CMS 운영 | 운영자 시스템 7개 모듈 콘텐츠 관리 대행 |
| 온라인 광고 | 네이버·구글·브랜드검색·배너·유튜브·SNS 바이럴 통합 운영 |
| 보고 | 월별 운영 보고서 + 광고 성과 리포트 |

## 3. 수행 방법론

### 3.1 운영 거버넌스

```
클라이언트 요청
      ↓
PM 접수 및 분류 (Critical/Major/Minor)
      ↓
      ├─ Critical: 즉시 대응 → 4h 내 조치
      ├─ Major: 당일 착수 → 3일 내 완료
      └─ Minor: 월별 배치 처리
      ↓
스테이징 개발 → 검증 → 프로덕션 배포
      ↓
클라이언트 완료 확인
```

### 3.2 월별 운영 사이클

| 주차 | 활동 |
|------|------|
| 1주차 | 전월 이슈 마감, 이달 요청사항 접수 및 우선순위 결정 |
| 2주차 | 주요 결함·기능 개발, 광고 캠페인 중간 점검 |
| 3주차 | 스테이징 검증 → 프로덕션 배포, 보안 패치 확인 |
| 4주차 | 월별 보고서 작성, GA4 성과 분석, 다음 달 계획 수립 |

### 3.3 온라인 광고 운영 방법론

**타깃 세그먼트 정의**
- 1차 타깃: 50~70대 본인 사전 준비 고객 (장례 정보 탐색)
- 2차 타깃: 30~50대 가족 준비 고객 (부모님 장례 준비)
- 긴급 타깃: 즉시 장례 서비스 필요 고객 (리마케팅)

**채널별 운영 전략**

| 채널 | 전략 | KPI |
|------|------|-----|
| 네이버 검색광고 | 브랜드명·서비스명 키워드 방어, 경쟁사 키워드 공략 | CPC, 전환율 |
| 구글 광고 | 디스플레이 리마케팅, 유사 타깃 확장 | CTR, ROAS |
| 브랜드 검색 | 브랜드 신뢰도 강화, 검색 점유율 유지 | 브랜드 검색량 |
| 배너광고 | 시즌별 상품 프로모션 소재 운영 | 노출 수, CTR |
| 유튜브 | 브랜드 스토리 영상, 서비스 설명 콘텐츠 | 조회율, VTR |
| SNS 바이럴 | 월 8편 이상 콘텐츠, 신뢰 중심 정보성 포스팅 | 도달 수, 저장 수 |

**월별 성과 관리 프로세스**
1. GA4 데이터 수집 (채널별 유입·전환·이탈)
2. 광고 플랫폼 성과 취합 (CTR·CPC·ROAS·전환수)
3. 저성과 키워드·소재 교체
4. 월별 광고 성과 보고서 제출

### 3.4 장애 대응 체계

**장애 등급 및 SLA**

| 등급 | 기준 | 대응 목표 |
|------|------|----------|
| Critical | 서비스 전체 중단 / 개인정보 침해 | 즉시 인지, 4시간 내 복구 |
| Major | 주요 기능 장애 (로그인·결제·신청) | 4시간 내 인지, 24시간 내 복구 |
| Minor | 일부 기능 오류, UI 깨짐 | 2영업일 내 처리 |

**모니터링 항목**
- 서버: CPU 80% 이상, 메모리 85% 이상, 디스크 90% 이상 → 경보
- 응답시간: 3초 초과 → 경보 (PER-001)
- 에러율: 5분 내 에러 50건 이상 → Critical 경보
- SSL 인증서: 만료 30일 전 → 갱신 착수
