# Spec: 이노빌드랩 유지보수/운영 제안서

## Overview
이노빌드랩(innobuildlab.com, IT/소프트웨어 기업) 대상 신규 수주 영업용 유지보수·운영 제안서.
RFP 없이 제안사가 먼저 접근하는 구조이며, 운영 계약 체결 후 기획 산출물을 순차 납품하는 방식으로 설계한다.

## Goals
- 이노빌드랩과 월 100~300만원 규모의 유지보수/운영 계약 수주
- 장기 파트너십으로 이어지는 관계 구축 (향후 리뉴얼·구축 프로젝트 선수주)
- 운영 계약 이후 기획 산출물 순차 납품으로 서비스 가치 가시화

## Scope

### In
- 유지보수/운영 제안서 본문 (A4 기준 10~15페이지 내외)
- 운영 서비스 범위 정의 (기능 개선/추가 중심)
- 납품 산출물 목록 및 일정 (운영 계약 후 월별 순차 납품)
  - 요구사항 정의서
  - IA / 사이트맵
  - 화면 설계서 (와이어프레임)
  - WBS / 일정계획

### Out
- 사이트 리뉴얼·구축 제안 (현 단계에서 제외, 추후 확장)
- 서버/호스팅 운영 (핵심 서비스 아님)
- 콘텐츠 업데이트 정기 서비스

## Inputs and Outputs

- **Input**: 이노빌드랩 공개 사이트(innobuildlab.com), 담당자 인터뷰 없음 (RFP 없음)
- **Output**: 유지보수/운영 제안서 (PDF 또는 PPT), 운영 착수 후 단계별 납품 산출물

## Requirements

### Functional (제안서 구성 요소)
- 현황 분석: 이노빌드랩 사이트 현재 상태 파악 및 개선 포인트 도출
- 서비스 제안: 기능 개선/추가 중심의 운영 서비스 정의 (월 단위 업무 범위 명시)
- 납품 산출물 계획: 요구사항 정의서·IA·와이어프레임·WBS를 월별로 납품하는 로드맵 포함
- 견적/계약 구조: 월 100~300만원 범위에서 티어 구성 또는 단일 요금제 제시
- 차별화 섹션: 장기 파트너십 가능성 (리뉴얼 프로젝트 연계, 전담 PM 채널 운영)

### Non-functional
- 제안사 포지션: 중소 에이전시(5~30인), 빠른 대응·합리적 비용을 강점으로 부각
- 기술 친숙도: 클라이언트 사이트가 React/Vue/Node 등 자체 개발 기반 → 동일 스택 운영 역량 명시
- 차별화 대응: 대형 에이전시와의 직접 비교보다 "밀착 파트너" 포지셔닝으로 차별화
- RFP 없음 보완: 사이트 직접 분석을 통해 클라이언트 니즈를 스스로 정의하여 설득력 확보

## Technical Notes
- 이노빌드랩 사이트 기술스택: 자체 개발 (React/Vue/Node 추정, SSL 인증서 이슈 확인됨)
- 사이트 접근: SSL 오류로 직접 크롤링 불가 → 공개 정보(SNS, 링크드인, 포트폴리오) 활용하여 현황 파악 필요
- 납품 산출물은 기존 `아데오 에이전트 시스템/templates/` 의 템플릿 활용

## Proposal Structure (제안서 목차 안)
1. 제안 배경 및 목적
2. 이노빌드랩 사이트 현황 분석 (문제점 / 개선 기회)
3. 운영 서비스 범위 및 프로세스
4. 납품 산출물 로드맵 (월별 순차 납품 계획)
5. 견적 및 계약 구조
6. 제안사 소개 및 레퍼런스
7. 장기 파트너십 비전 (리뉴얼 연계 로드맵)

## Tradeoffs & Decisions
- **RFP 없음 → 자체 분석 선행 필요**: 클라이언트 정보가 제한적이므로 사이트 분석 + 업계 표준 운영 범위로 제안 구성
- **소규모 계약 → 납품물 범위 조율**: 월 100~300만원에 기획 산출물 4종을 포함하면 수익성 이슈 가능 → 산출물 납품을 부가 가치로 포지셔닝하거나 단계별 옵션으로 제시
- **차별화 어려움 → 파트너십 스토리**: 기능 스펙 비교보다 "함께 성장하는 IT 파트너" 내러티브로 승부

## Failure Modes
- 사이트 분석 정보 부족 → 제안 범위가 너무 일반적이 될 수 있음: 대응책 = 제안서에 "킥오프 미팅 후 범위 정밀화" 조항 명시
- 월정액 가격 저항 → 옵션 A/B 티어 제공으로 진입 장벽 낮춤
- 기획 산출물 납품 부담 → 운영 착수 후 2개월 내 순차 납품, 추가 공수 없이 운영비 내 포함 구조로 설계
